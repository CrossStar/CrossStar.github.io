记录最近树莓派的环境配置

实在是闲的无聊，记录一下自己最近打比赛折腾树莓派 5 的经历。

Raspberrypi OS

  • 上手当然是直接烧录官方推荐的操作系统
    • 但是真的很坑！后面可能大多数问题都是这个操作系统引起的，虽然图形化操作树莓派的相关配置还是挺方便的

迫不及待配置好 SSH，VNC和基于 cpolar 的内网穿透后才开始界面的开发。

因为自己要在树莓派上运行基于 qfluentwifget 的 PyQt5 界面，所以自然要配置 PyQt 相关的环境…

首先用 pipvenv 创建一个虚拟环境，运行 pip install pyqt5，发现卡在 preparing metadata pyproject.toml 后报错，可能是 pip 版本不兼容？折腾半天后遂放弃

发现系统里面已经预装了 PyQt5，尝试用 pipx 下载其他包会报兼容性问题，卸载原来预装的版本后尝试了其他版本后仍然不行，放弃

（其实之前做信号实验也遇到过相同的问题，当时的方法是手动匹配包的版本号一个一个下 whl 文件，这次懒得折腾了）

最后决定还是用 miniconda，安装之后用 conda install pyqt 后就成功了！安装其他要的各种包后运行主程序，发现找不到 wayland 插件？上网找了半天原因后用 conda install qt-wayland 就好了。但是…诶，我左边那个基于 QWebEngineView 和 Echarts 的折线图怎么没加载出来？

  • 尝试修改 html 文件路径后无果，或者修改为 self.browser.setUrl(QUrl("https://www.baidu.com/")) 同样不行
  • 尝试 app = QApplication(sys.argv+['--disable-seccomp-filter-sandbox']) 后无果
  • 最后发现树莓派论坛上也有人有类似的问题，最后的解决方案是把操作系统换为 Ubuntu？？？那我之前配置的东西咋办？

于是重装系统，使用系统 Ubuntu 24.04.1 LTS

Ubuntu

首先得装上 SSH，因为 Ubuntu 没有预装 openssh-server 所以还得自己手动装一个，不过也不难，然后把 VSCode 和 XShell 之类的软件连上去就好了。

吸取教训，配置环境首先先把 miniconda 装上，然后装上各种要的包，运行主文件，界面很顺利就出来了？（为什么这里没有报找不到 wayland 插件的问题我也不知道，可能是原来的系统预装的 Qt 会自动把插件的环境变量配置到奇奇怪怪的地方？）

但是现在左边的折线图同样没有渲染出来，看了看 html 路径没问题之后再次尝试 app = QApplication(sys.arg +['--disable-seccomp-filter-sandbox']) 就成功了！右边用 FIFOCache 做的流式绘图好像比原来 ROS 系统上的还要顺滑一些？不知道是什么原因。

但是如果要远程开发的话还得能在本地电脑能显示界面才行，如果直接在 VSCode上运行界面程序会报错 xcb 插件之类的有问题，上网查了一下可能是 X Server 的原因，在 XShell 中运行让 XManager 转发就好了。但是好像效果不太理想，那个流式绘图界面在本地电脑上卡卡的，也太丑了。

果然还是得用 VNC 弄远程桌面。装上 aarm64 的 realvnc-vnc-server 之后发现怎么都打不开服务程序？

  • 重启后无果
  • 尝试换为 tigervnc-standalone-server 后，运行 vncserver 命令行卡死
    • 装上树莓派的设置包，运行 sudo raspi-config，在界面里尝试打开 VNC 也会报错，看看命令行的消息还是得用 RealVNC 才行（但这个应该不是根本原因）
  • 又翻遍树莓派论坛，找了几天用各种有关 Ubuntu，VNC，Raspberrypi 关键词，最后终于找到可能得禁用 wayland 才行
    • 运行 sudo nano /etc/gdm3/custom.conf，把 WaylandEnable=false 取消注释，然后常规运行各种类似开机启动的命令，最后重启就好了！（这次的远程桌面同样要比原来的系统更丝滑一些的说

到这里树莓派基本配置大概就折腾完了，内网穿透好像也不太需要因为校园网也可以直接 SSH 到的来着。

（话说最近也在弄和 PyTorch 相关的环境，但是好像远远没有这个难配，用 nvidia-smi 查看一下自己的 CUDA 版本号在官网上用 conda 下载对应版本的就好了，一路下来也挺顺利的）
